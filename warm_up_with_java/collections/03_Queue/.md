### 1. Dequeue: Queue follows FIFO
A Double Ended Queue (short: Deque, pronounced “deck”) is a data structure where you can add and remove elements from both ends.
#### Why use Deque?
= Use when you need:
= normal queue behavior (FIFO)
= stack behavior (LIFO)
= both in one structure
#### Common implementation
`Deque<String> dq = new ArrayDeque<>();`

#### Important methods
1. Front side:
- `addFirst(x)`, `offerFirst(x)`
`removeFirst()`, `pollFirst()`
`getFirst()`, `peekFirst()`
2. Back side:
- `addLast(x)`, `offerLast(x)`
`removeLast()`, `pollLast()`
`getLast()`, `peekLast()`
- Safe versions are offer/poll/peek (no exception on empty).


  
#### Core Queue methods
1) `offer(x)` -> add to queue
2) `poll()` -> remove and return front item (returns null if empty)
3) `peek()` -> see front item without removing
4) `isEmpty()`, `size()`
#### Use:
`Queue<String> q = new LinkedList<>();`

### PriorityQueue
PriorityQueue does not serve in normal FIFO order.
- It serves by priority:
- default: smallest element first (min-heap)
- duplicates allowed
- does not keep insertion order
- main methods same idea: offer, peek, poll


---
### Deque as Stack and Queue
1. As stack (LIFO):
    - push(x) / pop() / peek()
2. As queue (FIFO):
    - offerLast(x) / pollFirst()
